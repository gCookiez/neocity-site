The One Page Webpage works like a charm in my local machine. It is able to redirect to appropriate directories on url change.

However, the mechanics for neocities works differently as it tries to look for an .html page instead of fetching through my router.

Essentially, I made the page rely on json files to update the contents of the page wihtout needing to add html files. The HTML and it's assets acts like a shell for things.

As a side effect, when adding new paths to the url directly, then the servers in neocities sees it as an invalid path and redirects to the not_found.html page

I also found out that if that file is non existent in your paths, then a fallback "404" page is provided by the servers.




Fortunately, I was able to resolve the issue by injecting a script to that specific file. Neocities allows me to modify my own "404" page

The page was only an html page, so I added a script that gets the path of the link, packages it as a query with some URI modifications, redirects pack to the main page and goes through a checkpoint when the url contains queries

Queries are then cleared and uses the pushState API to then go thorugh the normal router routine.

This was a fascinating fix for the problem i had because I was able to pickup some of the inner workings of Neocities' services and how to cleverly pass parameters without any interruptions to these services.
